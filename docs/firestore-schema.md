# DistraKill Firestore Schema

This document defines the database structure for the DistraKill student-focus mobile application.

## 1. Users Collection

### Collection path

```text
users/{userId}
```

### Document ID

Use the Firebase Authentication user UID.

### Fields

| Field | Type | Required | Description |
|---|---|---|---|
| fullName | string | Yes | Student’s full name |
| email | string | Yes | Student’s email address |
| preferredTheme | string | Yes | `light` or `dark` |
| photoURL | string or null | No | Profile image URL |
| createdAt | timestamp | Yes | Account creation time |
| updatedAt | timestamp | Yes | Last profile update time |

### Example

```json
{
  "fullName": "Madina Akbari",
  "email": "madina@example.com",
  "preferredTheme": "light",
  "photoURL": null
}
```

## 2. Tasks Collection

### Collection path

```text
tasks/{taskId}
```

### Document ID

Automatically generated by Firestore.

### Fields

| Field | Type | Required | Description |
|---|---|---|---|
| userId | string | Yes | Firebase UID of the task owner |
| title | string | Yes | Task title |
| subject | string | Yes | Subject name |
| description | string | No | Additional task details |
| estimatedMinutes | number | Yes | Planned study duration |
| dueDate | timestamp or null | No | Task deadline |
| status | string | Yes | `pending`, `inProgress`, or `completed` |
| createdAt | timestamp | Yes | Task creation time |
| updatedAt | timestamp | Yes | Last task update time |

### Example

```json
{
  "userId": "firebase-user-id",
  "title": "Study Chapter 4",
  "subject": "Mathematics",
  "description": "Review algebra exercises",
  "estimatedMinutes": 45,
  "dueDate": null,
  "status": "pending"
}
```

## 3. Focus Sessions Collection

### Collection path

```text
focusSessions/{sessionId}
```

### Document ID

Automatically generated by Firestore.

### Fields

| Field | Type | Required | Description |
|---|---|---|---|
| userId | string | Yes | Firebase UID of the session owner |
| taskId | string or null | No | Related task document ID |
| plannedMinutes | number | Yes | Planned session duration |
| completedMinutes | number | Yes | Actual focused minutes |
| startedAt | timestamp | Yes | Session start time |
| endedAt | timestamp or null | No | Session end time |
| status | string | Yes | `active`, `completed`, or `cancelled` |
| createdAt | timestamp | Yes | Session creation time |

### Example

```json
{
  "userId": "firebase-user-id",
  "taskId": "task-document-id",
  "plannedMinutes": 25,
  "completedMinutes": 25,
  "status": "completed"
}
```

## 4. Duaas Collection

### Collection path

```text
duaas/{duaaId}
```

### Document ID

Automatically generated by Firestore.

### Fields

| Field | Type | Required | Description |
|---|---|---|---|
| title | string | Yes | Dua title |
| category | string | Yes | `stress`, `exam`, `forgetfulness`, or `general` |
| arabicText | string | Yes | Original Arabic text |
| translation | string | Yes | Translated meaning |
| transliteration | string | No | Pronunciation support |
| language | string | Yes | Translation language |
| source | string | No | Quran or Hadith source |
| isActive | boolean | Yes | Controls whether the dua is displayed |
| createdAt | timestamp | Yes | Creation time |

### Example

```json
{
  "title": "Dua for increasing knowledge",
  "category": "exam",
  "arabicText": "رَبِّ زِدْنِي عِلْمًا",
  "translation": "My Lord, increase me in knowledge.",
  "transliteration": "Rabbi zidni ilma",
  "language": "English",
  "source": "Quran 20:114",
  "isActive": true
}
```

## Collection Relationships

```text
Firebase Authentication User
        |
        ├── users/{userId}
        |
        ├── tasks/{taskId}
        |      └── userId
        |
        └── focusSessions/{sessionId}
               ├── userId
               └── taskId

duaas/{duaaId}
    Shared read-only content for authenticated users
```

## Security Summary

- Each user can access only their own user profile.
- Each user can create and manage only their own tasks.
- Each user can create and manage only their own focus sessions.
- Authenticated users can read duas.
- Regular app users cannot edit or delete duas.
